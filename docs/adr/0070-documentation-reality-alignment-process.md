# 0070 - Documentation Reality Alignment Process

## Status

Accepted

## Context

The audit found mild drift between the product framing and what a stranger can really do.

## Decision

README and in-app copy must describe only tested workflows. New claims require either a unit or e2e assertion, or a direct audit row proving the behavior.

## Consequences

Documentation becomes a verification artifact instead of aspiration.

## Alternatives Considered

Keeping broader product language was rejected because it creates support debt and user confusion.
